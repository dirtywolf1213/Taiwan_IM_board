# 用 AI 協助撰寫詳解:教學與指令

本文件給「協助撰寫題目詳解的 AI」(可用任何夠強的模型:Claude、ChatGPT、Gemini 等)及操作者使用。
目標:讓 AI 產出**格式統一、可審核、可直接入庫**的詳解草稿。

> 規範本體見 [`詳解撰寫規範.md`](./詳解撰寫規範.md);本文是「**怎麼用 AI 跑這套規範**」的操作手冊與現成指令。

---

## 一、整體流程(操作者)

```
① 匯出題目  →  ② 把「指令 + 題目」貼給 AI  →  ③ 取回 JSON
        →  ④ 貼進 explanations.<年>.json  →  ⑤ npm run validate
        →  ⑥ 醫師審核,正確者把 status 改成 "reviewed"
```

### ① 匯出題目
用內建工具匯出某年/某科「尚無詳解」的題目(輸出可直接貼給 AI):
```bash
python3 tools/export_questions.py --year 114 --subject 心臟血管
# 或指定題號:python3 tools/export_questions.py --year 114 --nums 1,11
```

### ② 餵給 AI
把下方〈**可複製的 AI 指令**〉整段貼給 AI,接著貼上①匯出的題目 JSON。
建議**一次 5–15 題**,品質與穩定度較好。

### ③④ 取回並入庫
AI 會回傳一個 JSON 物件(以題目 id 為 key)。把它**合併**進 `src/data/explanations.<年>.json`
(同一檔案、不要覆蓋既有的其他題)。

### ⑤ 驗證
```bash
npm run validate      # 會檢查 JSON 正確性,並報告各年詳解覆蓋率
```

### ⑥ 醫師審核(必要)
逐題確認醫學內容與引用無誤後,把該題的 `"status": "draft"` 改成 `"reviewed"`。
App 只對 `reviewed` 不顯示「草稿」標記。**未審核前一律維持 draft。**

## 二、可複製的 AI 指令(Prompt)

> 直接整段複製給 AI,然後在後面貼上要寫的題目 JSON。

````text
你是台灣「內科專科醫師甄審」考古題的詳解撰寫者。請為我提供的每一題,寫一則符合下列規範的詳解。

【輸入】
我會在最後給你一個 JSON 陣列,每個物件是一題,欄位:
  id, year, num, subject, question, options(A–E 字串陣列), answerLetter(正解字母)

【輸出 — 嚴格遵守】
只輸出「一個 JSON 物件」,不要任何多餘文字、不要 markdown 圍欄。以題目 id 為 key:
{
  "<id>": { "text": "<Markdown 詳解字串>", "status": "draft" }
}
- status 一律為 "draft"。
- 多題就放多個 key。
- text 內的換行用 \n,確保是合法 JSON 字串。

【深度要求 — v2(重要)】
寫到 board/專科考高度,不是複習卡。每題要帶**機轉、流行病學、鑑別推理、數值/cutoff、分情境決策**;錯的選項要點出「它是哪個概念的陷阱/混淆」。標竿範例見 114-087。仍用自己的話改寫,不整段抄書。

【text 必須是固定「五段式」,每段標題用 Markdown ### 】
### 本題觀念
點出核心概念,並說明**本題情境/數據如何指向診斷**(如 CSF 四格鑑別、影像/實驗室判讀)。
### 選項分析
逐一分析 A–E,用「- 」條列,每項標「正確 / 錯誤」+ 理由(1–3 句:機轉 / 流行病學 / 為何是經典陷阱)。正解標「**→ 本題答案**」並加粗。
### 答案解析
回扣題幹問法(問何者正確/錯誤/不是),總結為何選該答案,一到三句。
### 核心知識點
可帶走、可背且**可操作**的重點:**藥名與劑量、cutoff、分情境決策(依年齡/危險因子/部位)**、口訣。條列、關鍵字加粗。可補高產相關考點。
### 參考資料
2–4 筆真實來源。**時效規則**:附該主題**最新版指引(優先 2023–2026)+ ≥1 關鍵試驗/系統回顧**;下筆前用 WebSearch 確認有無更新版。指引若確實未改版,**仍列出並標「(現行版,尚未改版)」**。依型別給足欄位:
- 期刊/指引:`作者 et al. 標題. *期刊縮寫*. 年;卷(期):頁. PMID: xxxxxxxx. DOI: 10.xxxx/xxxx.`(有 PMID/DOI/官方網址者都要列)
- 教科書:`*書名*. 版次 ed. 出版者; 出版年（章節）.`(務必含版次與年份,如 Harrison's 21st ed., 2022)
- 政府/學會文件(CDC/WHO/台灣疾管署/ACIP):`發行單位. 標題（年/版本）. 官方網址`
期刊名用 *斜體*。PMID/DOI 必須是 WebSearch 實際查到的真值,**嚴禁憑記憶捏造**;查不到的欄位就留空不填。

【規則】
- 語言：繁體中文為主,專業名詞保留英文(如 LVEF、Class I、DOAC、ARNI、CHA₂DS₂-VASc)。
- 數值用 ≤ ≥;單位統一(mg/dL、mmHg、bpm)。
- 醫學正確性第一,寧缺勿錯。不確定的具體數字、試驗名稱不要寫,改成原則性描述。
- 嚴禁杜撰參考文獻：只列你確定真實存在的指引/期刊。查證時以最新版優先(2024–2026)。
  若你具備網路搜尋能力,先搜尋查證指引最新版本與引用,再下筆。
- 「正解」依該考試年當時的標準判定。若該考點的指引在考後有更新,於適當處用「> 」引述塊加一句
  「最新進展：…」說明差異,但正解仍依考年標準。
- 不要整段複製指引或教科書「內文」；一律用自己的話精簡改寫(避免侵權)。但**參考資料的書目資訊要完整精確**(作者/標題/期刊/年卷頁/PMID/DOI/網址)——書目不算侵權,要給足。
- 長度：完整但精簡,選項分析每項 1–2 句。

【格式範例(僅供參考結構,實際內容依題目而定)】
{
  "114-011": { "text": "### 本題觀念\n非瓣膜性心房顫動(AF)的中風預防,核心是用 **CHA₂DS₂-VASc** 評估風險、以 **DOAC** 為首選。\n\n### 選項分析\n- **(A) 不需抗凝** — 錯誤。CHA₂DS₂-VASc 高分屬高風險,需抗凝。\n- **(D) DOAC** — **正確 → 本題答案**。非瓣膜性 AF 首選,顱內出血少、不需監測 INR。\n（其餘選項略）\n\n### 答案解析\n高風險且為非瓣膜性 AF → 首選 DOAC,故選 (D)。\n\n### 核心知識點\n- CHA₂DS₂-VASc：男性 ≥2、女性 ≥3 建議抗凝。\n- 瓣膜性 AF(機械瓣、中重度二尖瓣狹窄)仍用 warfarin,禁用 DOAC。\n\n> **最新進展**：2024 ESC 改用 CHA₂DS₂-VA(移除性別項),結論一致。\n\n### 參考資料\n1. Van Gelder IC, Rienstra M, Bunting KV, et al. 2024 ESC Guidelines for the management of atrial fibrillation. *Eur Heart J*. 2024;45(36):3314-3414. PMID: 39210723. DOI: 10.1093/eurheartj/ehae176.", "status": "draft" }
}

準備好後,以下是題目：
（在這裡貼上 export_questions.py 匯出的 JSON）
````

## 三、入庫後的品質檢查清單

- [ ] `npm run validate` 通過(JSON 合法、覆蓋率有更新)。
- [ ] 五段齊全(本題觀念 / 選項分析 / 答案解析 / 核心知識點 / 參考資料)。
- [ ] 選項分析**每個選項都有**,正解標示「→ 本題答案」。
- [ ] 結論與題庫的 `answerLetter` **一致**(AI 偶爾會解錯方向,務必核對)。
- [ ] 參考文獻**真實存在**、版本/年份正確(重點防杜撰)。
- [ ] 參考文獻**精確完整**:期刊列出 PMID/DOI、教科書註明版次與年份、政府文件附官方網址(凡存在者皆列)。
- [ ] 涉及台灣健保給付者,有註明「以台灣現行規範為準」。
- [ ] 醫師已確認 → `status` 改 `reviewed`。

## 四、常見錯誤(審核時特別留意)

1. **答錯方向**:題目問「何者錯誤/不是」,AI 卻照「何者正確」解 → 結論與答案不符。
2. **杜撰引用**:列出看似合理但不存在的文獻或年份。
3. **過時**:沿用舊指引而非最新版(或反之,未標注考年與最新差異)。
4. **抄錄原文**:整段複製指引/教科書(品質與版權皆不宜)。
5. **JSON 壞掉**:換行未用 `\n`、多了圍欄或解說文字 → 無法入庫。`npm run validate` 會抓出。
