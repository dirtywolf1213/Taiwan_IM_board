# 詳解撰寫規範

本規範定義題目詳解的**統一格式、語氣與品質要求**,確保人工或 AI 產出穩定一致、可審核。
所有詳解一律遵循本規範。

> **維護原則(給人類與 AI)**:本規範與 [`AI寫詳解教學與指令.md`](./AI寫詳解教學與指令.md) 是「給協作 AI 的單一真實來源」。
> **任何原則性的調整**(格式、引用標準、查證流程、品質要求等)**一旦定案,都要同步更新這兩份文件**,
> 並在文末〈修訂紀錄〉補一條,讓後續任何 AI / 人都依最新規範作業。

---

## 0. 存放與狀態

- 詳解存於 `src/data/explanations.<年>.json`,以**題目 id**(如 `114-001`)為 key,**不寫進題庫 JSON**(避免重轉檔被覆寫)。
- 欄位:
  ```json
  { "114-001": { "text": "（Markdown 內文）", "status": "draft" } }
  ```
- `status`:
  - `draft` — AI 起草或未經醫師審核(App 顯示「草稿・未經審核」標記)。
  - `reviewed` — 已由醫師逐字審核確認(App 正常顯示,無標記)。
- **流程:AI 起草 → 醫師審核 → 改為 `reviewed`**。未審核前一律 `draft`。

## 1. 固定五段式結構

每則詳解依序包含下列五段,標題用 Markdown `###`:

```
### 本題觀念
### 選項分析
### 答案解析
### 核心知識點
### 參考資料
```

> **深度標準(v2)**:詳解應寫到「**board / 專科考**」高度,不是快速複習卡。每題要帶出**機轉、流行病學、鑑別推理、數值/cutoff、分情境決策**;但仍用自己的話精簡改寫,不整段抄教科書。標竿範例見 `explanations.114.json` 的 `114-087`(細菌性腦膜炎)。

### ① 本題觀念
- 一段話點出**這題在考什麼核心概念**,給讀者定位。
- 帶出關鍵架構或定義,並交代**本題情境/數據如何指向診斷**(如 CSF 四格鑑別、影像/實驗室判讀)。

### ② 選項分析
- **逐一**分析每個選項(A–E),用條列:
  ```
  - **(A) 選項重點** — 正確/錯誤。為什麼(機轉 / 流行病學 / 為何是經典陷阱)。
  ```
- **正解選項**要標明「**→ 本題答案**」並加粗。
- 重點在「**為什麼對 / 為什麼錯**」:錯的選項要點出**它是哪個概念的混淆/陷阱**(如「紫斑應對應 N. meningitidis 而非 H. influenzae」);對的選項補一句**支持證據或數據**。不是重述選項文字。

### ③ 答案解析
- 回扣題幹問法(問「何者正確/錯誤/不是」),總結為什麼選該答案。
- 一到三句即可。

### ④ 核心知識點
- 可帶走、可背的**高效重點**,且要**可操作**:具體**藥名與劑量、cutoff、分類、分情境決策(依年齡/危險因子/部位)**、口訣。
- 用編號或條列,關鍵字加粗(如「ceftriaxone 2 g Q12h + vancomycin;≥50 歲加 ampicillin」、「dexamethasone 0.15 mg/kg Q6h,須在首劑抗生素前/同時給」)。
- 適度補充**高產相關考點**(同題系列常一起考的鑑別/併發症/最新進展)。

### ⑤ 參考資料
- **時效規則(v2,重要)**:每題**至少附該主題「最新版」指引**(優先 **2023–2026**),並**加 ≥1 篇關鍵試驗 / 系統性回顧**;通常 **2–4 筆**。
  - 下筆前**務必用 WebSearch 確認該指引是否已有更新版**(常見更新較快:Sepsis、CAP、AMR、HF、AF、DM、疫苗排程)。
  - 若該主題的權威指引**確實尚未改版**(如 IDSA 腦膜炎 2004、SSTI 2014),**仍要列出並明確標注「(現行版,尚未改版)」**,避免讀者誤以為漏查。
  - 盡量同時並列**「最新指引 + 奠基性 RCT/landmark trial」**(例:腦膜炎 → ESCMID 2016 + de Gans NEJM 2002 dexamethasone)。
- **引用要夠精確、可溯源**(不能只寫「Harrison's, Pneumonia」這種粗略寫法)。依來源型別給足下列欄位:
  - **期刊論文 / 學會指引**:`作者(et al). 標題. *期刊縮寫*. 年;卷(期):頁. PMID: xxxxxxxx. DOI: 10.xxxx/xxxx.`
    凡有 **PMID、DOI、官方網址**者**都要列出**。
    例:`Metlay JP, et al. Diagnosis and Treatment of Adults with Community-acquired Pneumonia (ATS/IDSA). *Am J Respir Crit Care Med*. 2019;200(7):e45-e67. PMID: 31573350. DOI: 10.1164/rccm.201908-1581ST.`
  - **教科書**:`編者, eds. *書名*. 版次 ed. 出版者; 出版年（章節）.`,**務必註明版次與出版年**。
    例:`Jameson JL, et al., eds. *Harrison's Principles of Internal Medicine*. 21st ed. McGraw Hill; 2022（Tuberculosis）.`
  - **政府 / 學會網站文件**(CDC、WHO、台灣疾管署、ACIP 等):`發行單位. 標題（年/版本）. 網址`,**附官方網址**。
    例:`衛生福利部疾病管制署。猴痘 Mpox。https://www.cdc.gov.tw/`
- **來源的時效性與層級、禁用來源、防止杜撰引用,詳見 §5 文獻與證據來源。**

## 2. 語言與用詞

- **繁體中文為主,專業名詞保留英文**(符合台灣醫界習慣):Hb、LVEF、ARNI、Class I、SGLT2i 等不硬翻。
- 首次出現的縮寫可附全名,如 GDMT(guideline-directed medical therapy)。
- 數值與單位統一:mg/dL、mmHg、bpm、≤ ≥(用數學符號,不用 <= )。
- 中英之間不用刻意加空白(渲染自然即可);**藥名用學名**(Bisoprolol),必要時附中文。

## 3. Markdown 慣例

| 用途 | 寫法 |
|---|---|
| 段落標題 | `### 本題觀念` |
| 關鍵字 / 答案 / 藥名 / 數值 | `**粗體**` |
| 選項、要點 | `- ` 條列 |
| 有序步驟 / 清單 | `1. 2. 3.` |
| 期刊名 | `*European Heart Journal*` |
| 補充提醒 | `> 引用區塊` |

## 4. 醫學正確性與安全(最重要)

- **寧缺勿錯**:不確定的內容不寫;錯誤詳解比沒有詳解更糟。
- 以**最新指引 / 主流教科書(Harrison's、UpToDate、台灣健保規範等)** 為準;涉及台灣健保給付者註明「以台灣現行規範為準」。
- **不杜撰**具體數字、試驗名稱、劑量;不確定就寫原則性描述。
- AI 產出一律先 `draft`,**務必經醫師逐題審核**才改 `reviewed`。
- 內容僅供考試學習,**不作為臨床決策依據**(與全站免責聲明一致)。

## 5. 文獻與證據來源(引用標準)

詳解的醫學主張必須有可靠來源支持。撰寫(尤其 AI 起草)與審核時,依下列標準找文獻、選來源。

### 5.1 時效性(越新越優)
- **優先採用最新版本**:有 2025 / 2026 的指引或重要試驗就優先採用;其次取最近一版。
- 指引引用**最新版或最新 focused update**(例:HF 用 2021 ESC + 2023 ESC focused update + 2022 AHA/ACC/HFSA)。
- 教科書 / 資料庫用**最新版**(Harrison's 最新版、UpToDate 最新更新)。
- **考試答案 vs 最新知識(重要)**:題目「正解」是依**該考年當時的標準**判定。若之後指引更新導致建議改變,詳解應:
  1. 先依**該考年標準**說明為何此為正解(確保答對考題);
  2. 再以「**最新進展**」一句註明後續更新或差異(讓讀者知道現況)。

### 5.2 來源層級(由高到低)
1. **官方 / 學會指引**
   - 國際:ESC、ACC/AHA、ADA、KDIGO、IDSA、GOLD、GINA、ASCO、ASH 等該領域權威指引。
   - 台灣:各專科醫學會指引、衛福部 / 疾管署公告、**健保給付規範**(涉及給付一律以台灣現行規範為準)。
2. **頂尖期刊原始文獻**:NEJM、Lancet、JAMA,及各專科頂刊(JACC、Circulation、Blood、Gut、J Am Soc Nephrol、Ann Intern Med 等)。
3. **標準教科書 / 實證資料庫**:Harrison's、UpToDate、各專科標準教科書。
4. 其餘同儕審查文獻。

### 5.3 禁用 / 避免
- 掠奪性期刊、未經同儕審查內容、個人部落格、內容農場、無法溯源的二手資料。
- **嚴禁杜撰引用**:AI 常見「幻覺式」捏造不存在的文獻 / 作者 / 年份。**每一筆參考都必須真實可查**;不確定就不要列,改以原則性描述並標 `draft` 待審。

### 5.4 引用做法(精確度要求)
- 主張與來源要**對得上**(不要掛一個無關的指引)。
- **引用必須精確、可溯源**,不要只寫來源名稱。逐筆給足識別資訊:
  - 期刊 / 指引:**作者、標題、期刊、年;卷(期):頁**,並列出**所有可得的 PMID、DOI、官方網址**。
  - 教科書:**書名 + 版次 + 出版年 + 章節**(例:Harrison's 21st ed., 2022)。
  - 政府 / 學會文件:**發行單位 + 標題 + 年/版本 + 官方網址**。
- 標明**版本 / 年份**(如「2023 ESC HF focused update」)。
- 台灣在地議題(流行病學、給付、疫苗政策)用**台灣來源**(疾管署 / 各專科醫學會 / 健保署),附官方網址。
- 來源彼此衝突時,以較高層級 / 較新者為準,並簡短註明差異。
- **查證流程**:PMID / DOI / 版次 / 年份一律以 WebSearch 實際查證為準(見 §5.5),**不得憑記憶捏造 PMID 或 DOI**;查不到就不要硬填該欄位。

### 5.5 起草時的查證流程(工具與限制)
- AI 起草**應主動用 WebSearch(網路搜尋)查證**:確認指引最新版本與年份、是否有新版、引用是否真實存在。WebSearch 走伺服器端,**不受本專案沙箱對外網路限制**,可正常使用(與使用手機 / 電腦客戶端無關)。
- 限制:WebSearch 可取得結果的**標題、網址、摘要片段**;但**抓取網頁全文(WebFetch / curl)會被執行環境擋住**,無法逐字精讀整篇 PDF。故以「搜尋摘要 + 權威來源(指引 / 頂刊)」為準;若某具體數字無法查證,改寫成原則性描述,不杜撰。
- AI 訓練知識截止約 2026 初;**更新較快的領域以 WebSearch 查到的最新版為準**。
- 即使已查證,AI 草稿仍一律 `draft`;**最終由醫師確認來源與內容後**,才改 `reviewed`。

## 6. 長度

- **v2 深度導向**:寧可資訊密度高,也不要空泛。選項分析每項 **1–3 句**(含機轉/陷阱);核心知識點給**可操作**的劑量/cutoff/分情境決策。
- 仍以**自己的話精簡改寫**為原則,**避免長篇教科書式抄錄**(品質與版權皆不宜)。
- 標竿:`explanations.114.json` 的 `114-087`(細菌性腦膜炎)即 v2 深度與文獻寫法。
